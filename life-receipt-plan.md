# Life Receipt (인생 영수증) — 서비스 기획서

---

## 1. 컨셉 한 줄 요약

**"당신의 인생을 영수증 한 장으로 요약합니다"**

나이, 국가, 생활패턴을 입력하면 지금까지 살면서 소비한 시간·음식·행동을 편의점 영수증 스타일로 출력하는 바이럴 웹 서비스.

---

## 2. 입력 항목 설계

사용자 피로를 줄이기 위해 **필수 3개 + 선택 6개**, 총 15초 내 완료 목표.

### 필수 입력 (이것만으로도 영수증 생성 가능)
| 항목 | 입력 방식 | 용도 |
|---|---|---|
| 생년월일 | 날짜 선택 | 총 생존 일수 계산의 기준 |
| 국가 | 드롭다운 (IP 기반 자동 선택) | 국가별 통계 상수 적용 |
| 성별 | 2버튼 선택 | 통계 보정 |

### 선택 입력 (영수증 항목을 풍성하게)
| 항목 | 입력 방식 | 영향 |
|---|---|---|
| 하루 수면 시간 | 슬라이더 (4~12시간) | 수면 총량 |
| 하루 커피 잔수 | 0~10 숫자 선택 | 커피 소비량 |
| 하루 스마트폰 사용시간 | 슬라이더 (1~14시간) | 스크린타임 총량 |
| 식사 유형 | 3버튼 (집밥 / 반반 / 외식) | 식사 횟수 및 비용 |
| 반려동물 유무 | 토글 | 산책 시간, 사료비 등 |
| 직업 유형 | 4버튼 (학생/직장인/자영업/기타) | 출퇴근, 회의 시간 등 |

> 💡 선택 입력은 접힌 상태로 "더 정확한 영수증 받기 ▼" 로 노출.
> 입력 안 하면 국가별 평균값으로 자동 계산.

## 3. 영수증 항목 (출력)

편의점 영수증 포맷을 그대로 모방. 항목은 카테고리별로 구분선(--------)으로 나눔.

### 카테고리 A: 시간 소비
| 항목 | 계산 로직 (예시: 30세 기준) | 표시 형식 |
|---|---|---|
| 총 생존 시간 | 30 × 365.25 × 24 | 262,980시간 |
| 수면 | 생존일 × 수면시간 | 87,660시간 (10.0년) |
| 일/공부 | 직업유형별 × 근무년수 | 52,000시간 (5.9년) |
| 스마트폰 | 생존일(스마트폰 보급 이후) × 일사용시간 | 21,900시간 (2.5년) |
| 화장실 | 생존일 × 25분 (세계 평균) | 4,566시간 (190일) |
| 식사 | 생존일 × 1.5시간 | 16,436시간 (1.9년) |
| 이동/출퇴근 | 직업유형별 × 근무일 | 7,800시간 (0.9년) |
| 샤워/목욕 | 생존일 × 15분 | 2,739시간 (114일) |
| 대기/줄서기 | 국가별 평균 | 1,500시간 (63일) |
| SNS/유튜브 | 스마트폰 시간의 40% | 8,760시간 (1.0년) |
| **남은 자유시간** | **총시간 - 위 합계** | **??시간 (?.?년)** |

> 💡 마지막 "남은 자유시간"이 핵심 펀치라인. 의외로 적어서 충격 → 공유 동기 발생.

### 카테고리 B: 소비량
| 항목 | 계산 로직 | 표시 형식 |
|---|---|---|
| 커피 | 일 잔수 × 커피 마신 년수 | 10,950잔 |
| 식사 횟수 | 생존일 × 3 | 32,873끼 |
| 물 | 생존일 × 2L | 21,915L (욕조 109개) |
| 음식 무게 | 생존일 × 1.8kg | 19,724kg (코끼리 3마리) |
| 걸음 수 | 생존일 × 국가평균 보행 | 109,575,000걸음 |
| 머리카락 자란 길이 | 년수 × 15cm | 4.5m |
| 심장 박동 수 | 생존일 × 100,000 | 1,095,750,000회 |
| 눈 깜빡임 | 생존일 × 활동시간 × 15회/분 | 315,000,000회 |

> 💡 "코끼리 3마리", "욕조 109개" 같은 직관적 비유가 바이럴 포인트.

### 카테고리 C: 돈 (선택 — 소득 입력 시 활성)
| 항목 | 계산 로직 | 표시 형식 |
|---|---|---|
| 커피 지출 | 잔수 × 국가별 평균 커피 가격 | $16,425 |
| 식비 총액 | 식사횟수 × 국가별 평균 | $98,619 |
| 스마트폰 요금 | 사용년수 × 월평균 | $5,400 |
| 이발/미용 | 성별 × 국가별 평균 × 횟수 | $3,600 |

## 4. UI/UX 설계

### 4-1. 전체 흐름
```
[랜딩] → [입력 폼] → [로딩 애니메이션] → [영수증 결과] → [공유]
  1초      10~15초        2~3초              체류             바이럴
```

### 4-2. 랜딩 페이지
- 배경: 어두운 단색 (다크그레이 or 블랙)
- 중앙에 영수증 모양 미리보기 (모자이크 처리된 샘플)
- 카피: "Your life. One receipt." / "인생을 영수증 한 장으로"
- CTA 버튼: "내 영수증 받기" / "Get My Receipt"
- 아래에 공유된 영수증 수 실시간 카운터 (사회적 증거)

### 4-3. 입력 폼
- 한 페이지에 모두 표시 (스크롤 없이)
- 필수 3개가 먼저 보이고, 선택 항목은 접힌 아코디언
- 입력 방식은 최대한 탭/슬라이드 (키보드 입력 최소화)
- 모바일 퍼스트: 엄지 하나로 완료 가능하게

### 4-4. 로딩 애니메이션
- 영수증 프린터에서 종이가 출력되는 애니메이션
- "계산 중..." 문구와 함께 항목이 하나씩 타이핑되는 효과
- 의도적으로 2~3초 딜레이 (즉시 결과보다 기대감 조성)
- 프린터 "지지직" 사운드 효과 (옵션)

### 4-5. 결과 페이지
- 영수증 디자인을 최대한 사실적으로 구현
  - 흰색 배경 + 모노스페이스 폰트
  - 상단에 가게 로고 느낌의 서비스 로고
  - 하단에 바코드 (실제 바코드 아닌 장식용)
  - 영수증 특유의 점선 구분선 (---------)
  - 영수증 하단이 찢어진 듯한 지그재그 테두리
- 배경은 어둡게 → 영수증이 떠있는 느낌
- 각 항목에 "?" 아이콘 → 탭하면 계산 근거 표시 (신뢰도 확보)
- 스크롤 시 영수증이 위로 올라가는 느낌 (프린터에서 나오듯)

### 4-6. 공유 영역 (결과 하단)
- "내 영수증 공유하기" 버튼 그룹
  - 이미지 저장 (영수증을 PNG로)
  - 카카오톡 공유
  - 트위터/X 공유
  - 인스타그램 스토리 (이미지 저장 유도)
  - 링크 복사
- "친구 영수증 비교하기" → 링크 생성 (바이럴 루프)
- 공유 버튼 아래: "수명 계산기도 해보기" → 추후 확장 서비스 유도

## 5. 다국어 / 글로벌 전략

### 5-1. 언어별 페이지 구조
```
lifereceipt.com/         → 영어 (기본)
lifereceipt.com/ko/      → 한국어
lifereceipt.com/ja/      → 일본어
lifereceipt.com/zh/      → 중국어 간체
lifereceipt.com/es/      → 스페인어
lifereceipt.com/pt/      → 포르투갈어 (브라질)
```

### 5-2. 우선순위
| 순위 | 언어 | 이유 |
|---|---|---|
| 1 | 한국어 + 영어 | 동시 런칭. 한국에서 초기 바이럴 테스트 + 영어로 글로벌 기반 |
| 2 | 일본어 | 한국과 유사한 바이럴 문화, 점술/운세 콘텐츠 수요 높음 |
| 3 | 스페인어 | 사용 인구 5억+, 중남미 SNS 바이럴 활발 |
| 4 | 포르투갈어 | 브라질 시장 (인구 2억+, SNS 사용률 세계 최고 수준) |
| 5 | 중국어 | 위챗 공유 생태계, 다만 방화벽 이슈 고려 |

### 5-3. 로컬라이징 포인트
- **통계 상수**: 국가별로 다르게 적용 (커피 소비량, 평균 수면, 평균 식비 등)
- **통화**: 국가에 맞는 통화 기호 (₩, $, ¥, €, R$ 등)
- **비유 단위**: 문화권에 맞게 조정
  - 한국: "소주 OO병", "한강 OO번 왕복"
  - 미국: "football fields", "Olympic pools"
  - 일본: "도쿄타워 OO개 높이"
- **영수증 하단 문구**: 각 언어별 감성적 마무리
- **IP 기반 자동 언어 선택** + 수동 변경 가능

### 5-4. 번역 전략
- v1: 영어/한국어는 직접 작성
- v2: Claude API로 나머지 언어 번역 (비용 최소)
- 고정 UI 텍스트는 JSON i18n 파일로 관리
- 영수증 항목명은 짧아서 번역 품질 이슈 적음

## 6. 공유 / 바이럴 최적화

### 6-1. OG (Open Graph) 메타 태그
```
og:title       → "나는 지금까지 커피 12,045잔을 마셨다 ☕"
og:description → "당신의 인생을 영수증 한 장으로. 지금 확인해보세요."
og:image       → 동적 생성된 영수증 미리보기 이미지 (아래 참조)
og:url         → https://lifereceipt.com/r/abc123
```

### 6-2. 동적 OG 이미지 생성 ⭐ (핵심)
공유 링크를 카톡/트위터에 붙여넣었을 때 **영수증 미리보기**가 보여야 클릭률이 폭발.

**구현 방법:**
- 결과 생성 시 서버에서 영수증 이미지를 자동 생성 (Canvas API or Cloudflare Workers + @cloudflare/pages-plugin-vercel-og 스타일)
- 또는 Satori + Resvg 로 서버사이드 SVG → PNG
- 각 결과마다 고유 URL: `lifereceipt.com/r/{id}`
- 해당 URL 접근 시 OG 이미지는 생성된 영수증, 클릭하면 "나도 해보기" 랜딩으로 이동

**OG 이미지 디자인:**
- 영수증 상단 일부만 보여줌 (잘린 느낌 → 궁금증 유발)
- 핵심 항목 2~3개만 노출: "☕ 커피 12,045잔 / 🚽 화장실 190일 / 📱 스마트폰 2.5년"
- 하단: "나의 영수증 전체 보기 →"

### 6-3. 카카오톡 공유 최적화 (한국)
- 카카오 JavaScript SDK 연동
- 커스텀 카카오톡 공유 템플릿:
  - 제목: "OO의 인생 영수증"
  - 설명: "커피 12,045잔, 화장실 190일..."
  - 이미지: 동적 OG 이미지
  - 버튼: "나도 해보기"

### 6-4. 트위터/X 최적화
- Twitter Card: summary_large_image
- 자동 트윗 문구 프리셋:
  "나는 지금까지 인생에서 화장실에 190일을 보냈다 🧻 #LifeReceipt #인생영수증"

### 6-5. 인스타그램 스토리
- 인스타는 링크 공유가 안 되므로, **이미지 저장 → 스토리 업로드** 유도
- 영수증 이미지에 서비스 URL 워터마크 삽입
- 스토리 최적 비율(9:16)에 맞는 별도 이미지 템플릿

### 6-6. 결과 URL 구조
```
lifereceipt.com/r/{hash}     → 공유용 결과 페이지
```
- hash는 입력값을 인코딩한 짧은 문자열 (DB 불필요, 클라이언트에서 디코딩)
- 또는 쿼리스트링: `lifereceipt.com/r?a=30&c=KR&g=M&...`
- 서버리스로 OG 이미지만 동적 생성하면 DB/백엔드 없이 운영 가능

## 7. 기술 스택 (제안)

| 구성 요소 | 기술 | 이유 |
|---|---|---|
| 프레임워크 | Astro 또는 순수 HTML/JS | 정적 사이트, 빌드 빠름, Cloudflare Pages 최적 |
| 스타일링 | Tailwind CSS | 빠른 개발, 반응형 쉬움 |
| 호스팅 | Cloudflare Pages | 무료, 글로벌 CDN, 빠름 |
| OG 이미지 | Cloudflare Workers + Satori | 서버리스 동적 이미지 생성 |
| i18n | JSON 파일 기반 | 단순, 유지보수 쉬움 |
| 분석 | Cloudflare Analytics + GA4 | 무료, GDPR 친화적 |
| 공유 | 카카오 SDK + Web Share API | 한국 + 글로벌 공유 |
| 도메인 | Cloudflare Registrar | 도메인+호스팅+DNS 통합 관리 |

**대안: 순수 정적 사이트 (가장 단순)**
- 단일 HTML + JS + CSS
- 모든 계산은 클라이언트에서 수행
- OG 이미지만 Cloudflare Workers로 처리
- DB, 서버, API 일절 없음 → 유지비 0원

## 10. 핵심 성공 지표 (KPI)

| 지표 | 목표 (1개월) | 측정 방법 |
|---|---|---|
| 총 영수증 생성 수 | 100,000+ | GA4 이벤트 |
| 공유율 | 생성의 30%+ | 공유 버튼 클릭 / 생성 수 |
| 공유 → 재유입 전환율 | 20%+ | 공유 링크 통한 신규 방문 |
| 평균 체류 시간 | 90초+ | GA4 |
| 바이럴 계수 (K-factor) | 1.0 이상 | 공유율 × 전환율 |

> K-factor ≥ 1.0 이면 자연 성장 (사용자가 알아서 늘어남)
